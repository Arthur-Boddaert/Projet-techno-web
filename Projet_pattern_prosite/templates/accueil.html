Projet Pattern prosite

Description

Ce site à pour but de permettre l'exécution depuis un navigateur internet d'un script Python permettant de définir un pattern prosite à partir de plusieurs séquences d'acides aminés puis de rechercher si une autre séquence contient le pattern.

Pour cela il faut disposer d’un fichier au format FASTA contenant l’ensemble des séquences d’acides aminés, d’un second fichier FASTA contenant une autre séquence dont on veut vérifier la présence du pattern à l’intérieur et enfin d’un seuil correspondant au nombre maximum d’acide aminé différent toléré à une position avant de mettre un X dans le pattern.

Les scripts peuvent également être retrouvés ici.

Explication du fonctionnement des scripts

Le programme se base sur trois classes Python :
- La classe Multifasta permet de gérer toutes les séquences contenues dans le fichier FASTA, sous la forme d’une liste de chaînes de caractères.
- La classe MultipleAlignment permet d’obtenir des informations sur le nombre de séquences, savoir si une séquence contient des indels, si un site est conservé ou alors quels sont les acides aminés présents sur ce site.
- La classe PrositePattern permet la détermination du pattern prosite des séquences protéiques puis dans un deuxième temps il permet de vérifier si une nouvelle séquence protéique contient le pattern produit.

Pour commencer le script utilise les chemins d'accès pour extraire les séquences du fichier FASTA puis il utilise la deuxième classe pour pouvoir effectuer des actions sur les séquences est ainsi récupérer les informations voulues par l’utilisateur. Enfin la troisième classe est utilisée pour définir le pattern et effectuer la recherche du pattern dans une nouvelle séquence. Dans cette dernière classe seule la fonction de recherche n’est pas une fonction privée, contrairement aux autres classes où toutes les fonctions sont accessibles sans restrictions.

Pistes d'améliorations

Ce projet est un projet étudiant qui n’a pas été pensé pour être lancé depuis un navigateur web, il comporte donc divers points à améliorer du point de vue de l’intégration web ou de la partie programmation.
Divers points d’amélioration sont listés à la suite.

Pour le moment les scripts Python derrière ce programme, ont besoins des chemins d'accès vers les fichiers sur lesquels on souhaite lancer le programme. Une voie d'amélioration est la prise en charge des séquences au format Fasta copié directement dans le navigateur. Une autre option complémentaire serait que les utilisateurs puissent fournir directement les fichiers depuis le navigateur. Afin de faciliter l'accessibilité du programme.

Il faudrait modifier certains des scripts et en créer de nouveaux afin de pouvoir faciliter les calculs et ne pas effectuer trop de fois les mêmes opérations. Ajouter des fonctions permettant de retirer plus d’informations des séquences.
